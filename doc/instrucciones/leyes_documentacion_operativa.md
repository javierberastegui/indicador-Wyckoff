# Leyes de documentación operativa

1. La documentación viva forma parte del sistema.
2. Los logs por dominio son la bitácora principal.
3. `doc/logs/historico.md` no debe usarse como bitácora principal.
4. Cada dominio activo debe tener log y relevo cuando haya continuidad abierta.
5. Todo cambio debe registrar validación o indicar qué no se pudo validar.
6. Las incidencias reales se documentan en `doc/logs/incidencias/`.
7. No crear dominios duplicados si el mapa ya resuelve el área.
